UPWORK COMPLIANCE + LINK PLACEMENT
1) Profile + Specialized Profiles
- OK: link to portfolio/website as proof of work.
- NOT OK: “Email me to get started” / “Pay via Stripe” / “Book my Calendly instead of Upwork.”
- Placement: Portfolio section and Overview section as: “Portfolio/proof: [URL].” Do NOT position it as a way to contact or pay.
- Email: Only include the business email in a “Company details”/legitimacy context if Upwork allows AND wording does not instruct off-platform communication. Safer: omit email on Upwork until contract exists.

2) Project Catalog offers
- CTA language must be: “Order this Project” / “Message me on Upwork.”
- Requirements step: collect what we need inside Upwork (source link, topics, target audience, preferred style). After contract/order is placed, we can share Drive upload instructions.

3) Post-contract workflow (Upwork)
- After the client purchases or a contract is active, send:
  a) Tally intake link (optional) OR just paste the intake questions directly in Upwork chat.
  b) Drive upload instructions (folder naming + permissions).
  c) If a call is needed, propose scheduling inside Upwork messages; if allowed post-contract, share Calendly.

FIVERR COMPLIANCE + LINK PLACEMENT
1) Gigs
- Fiverr is strict about external links intended to move communication/payment off-platform.
- Use the website URL as “portfolio/proof” only (in description images/portfolio area if permitted). Avoid clickable external links in the gig description if Fiverr flags them.
- Do not place email publicly in the gig. Use Fiverr inbox.

2) After order is placed (Fiverr)
- Share Google Drive upload instructions and (if needed) a Calendly link only after the order is active, framed as a convenience for discussing the Fiverr order.

TALLY + CALENDLY USAGE RULE
- Primary: for leads coming from Carrd/organic (non-marketplace).
- Secondary: for marketplace clients only AFTER they’ve ordered (or when the platform explicitly allows pre-contract external scheduling, which is rare).
