Scope (DFY clip package default)
- Inputs: 1 long video/podcast episode (YouTube link, Drive link, or uploaded file)
- Output: 3–10 clips (9:16), each 20–60s unless otherwise specified
- Each clip includes: strong opening hook, burned-in captions, light punch-ins, basic cleanup, and export settings optimized for Reels/TikTok/Shorts

Client intake (must-have fields)
Collect via email or copy/paste form. Do not start production until these are answered:
1) Source link/file + timestamps if they have favorites
2) Platform priority (TikTok / Reels / Shorts) and handle(s)
3) Audience + offer (what they sell, who it’s for)
4) Caption style preference (clean / bold / word-by-word)
5) Brand kit (fonts/colors) if they have it; if not, use clean defaults
6) CTA preference (follow, comment keyword, link in bio, book call)
7) “Do not include” list (topics, words, claims)

Internal production timeline (24 hours)
T+0–1 hr: Intake + confirmation
- Confirm receipt and ETA.
- If anything is missing, request it immediately.
- Create a job folder: /ClientName/YYYY-MM-DD/EpisodeName/

T+1–3 hr: Content scan + clip plan
- Skim episode at 1.5–2x speed or use transcript (if available).
- Identify 5–15 candidate moments using these selection rules:
  - Clear standalone value in <60s
  - Strong contrarian point, step-by-step, mistake to avoid, or emotional story beat
  - Minimal context required (avoid heavy callbacks)
- Write a “clip plan” list:
  - Clip #, start/end time, 6–10 word hook, 1-sentence payoff, suggested on-screen headline

T+3–10 hr: Rough cuts + hook tightening
- Create rough vertical framing (9:16), keep subject centered.
- Add quick punch-ins / jump cuts to maintain pace.
- If the first 1–2 seconds are weak, prepend a hook line (text-only) or start later.

T+10–18 hr: Captions + polish
- Captions: burned-in, high contrast, safe-area compliant.
- Clean filler words only if it improves clarity; don’t over-edit authenticity.
- Optional b-roll cues (lightweight): add on-screen notes like [B-ROLL: “calendar”, “DMs”, “results screenshot”] if it materially increases clarity.

T+18–22 hr: QA pass + exports
- Ensure audio is consistent and understandable.
- Export settings (baseline):
  - 1080x1920 (9:16)
  - H.264, 10–16 Mbps
  - 30 fps (match source if needed)
  - AAC audio 320 kbps
- File naming convention:
  Client_Episode_Clip01_HookPhrase_9x16.mp4

T+22–24 hr: Delivery + next-step ask
- Deliver via Google Drive/Dropbox folder.
- Include a simple posting suggestion for each clip (title + caption starter + CTA).
- Ask for approval or 1 consolidated revision list.

Quality bar checklist (do not skip)
Visual
- Subject not cropped; eyes in upper third
- Captions not blocked by UI (TikTok/Reels safe areas)
- No distracting dead air at start/end
- Consistent style across clips (font, size, color)

Audio
- Voice clear; no clipping
- Background noise acceptable (basic cleanup if needed)

Content
- Hook present in first 1–2 seconds
- One clear idea per clip
- No risky claims (health/finance) unless client approved language

Revisions policy (scope guardrails)
- Included: 1 revision round per clip set (client sends one consolidated list).
- Revisions mean: trim, swap hook text, caption corrections, minor reframing.
- Not included: full re-editing of new angles, new clips beyond package count, heavy motion graphics, sourcing external b-roll libraries (unless client provides assets).
- If client requests major changes, respond with: “Happy to do it—can you prioritize the top 2 changes that matter most?” (keep work bounded).
