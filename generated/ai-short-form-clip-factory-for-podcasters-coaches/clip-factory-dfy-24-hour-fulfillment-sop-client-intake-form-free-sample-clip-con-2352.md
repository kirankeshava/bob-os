GOAL
Deliver 3–10 ready-to-post vertical clips from a long-form video/podcast within 24 hours (initial delivery), with captions + hooks + platform formatting. Minimize back-and-forth by collecting correct inputs upfront.

SERVICE PROMISE (client-facing)
“We turn one long video/podcast into ready-to-post Reels/TikToks/Shorts: strong hook, clean captions, punchy pacing, and a delivery folder you can post from. 24-hour first delivery. Reply to agent_bob_replit+clip-factory@agentmail.to if anything is missing.”

TOOLS (FREE/LOW-COST DEFAULTS)
- Transcription: YouTube auto-captions download or free transcription where available; if client provides transcript, use it.
- Editing: CapCut Desktop or DaVinci Resolve (free). Captions: CapCut auto-captions / built-in caption tools.
- Storage/Delivery: Google Drive folder or Dropbox (free tier).
- Docs/Tracking: Google Sheets (fulfillment tracker) + Google Docs for notes.

ROLES (even if one-person)
- Producer (intake + project setup)
- Editor (clip selection + edit)
- QA (check captions/safe areas/audio)
Note: One person can do all roles, but do QA as a separate final step.

STANDARD OUTPUT SPEC
- Format: 9:16 vertical
- Resolution: 1080x1920
- FPS: match source (or 30fps)
- Audio: -14 LUFS target (or normalize to avoid clipping); remove obvious background noise if possible
- Captions: burned-in; 2 lines max; high-contrast; highlight keywords (if supported)
- Safe areas: keep key text within platform-safe margins (avoid top/bottom UI)
- Hook: first 1.5 seconds must include movement or on-screen text hook
- Clip length guidelines:
  - TikTok/Reels: 15–45s (default)
  - YouTube Shorts: 20–50s (default)

FULFILLMENT WORKFLOW (24-HOUR SOP)
T+0: Intake received
1) Create a project folder: /ClientName_EpisodeDate/
   Subfolders: 01_Source, 02_Transcripts, 03_Selects, 04_Exports, 05_Thumbnails(optional), 06_Delivery
2) Confirm receipt (client message template below). Ask ONLY one follow-up question if critical info missing.

T+1 to T+3 hours: Source prep + transcript
3) Download source (or request client-provided file). If link-only, capture highest quality.
4) Generate transcript (YouTube captions export or in-editor auto captions). Save transcript file in 02_Transcripts.

T+3 to T+8 hours: Highlight selection
5) Identify 8–15 candidate moments. Use these heuristics:
   - Clear “teachable point” in one breath
   - Strong opinion/contrarian take
   - Step-by-step framework
   - Story with payoff
   - Specific numbers/results
6) Mark timecodes + one-line hook idea for each candidate in a simple selects doc (03_Selects/selects.doc).
7) Choose the best 3–10 depending on package.

T+8 to T+18 hours: Editing + captions
8) For each clip:
   - Start with the punchline or tension when possible (trim long preambles).
   - Remove filler words and dead space; add jump cuts.
   - Add on-screen hook text within first 1–2 seconds.
   - Add captions (burned-in). Fix proper nouns.
   - Optional: subtle b-roll cues in notes if you can’t source b-roll (example: “B-ROLL: screenshot of dashboard/graph”).
   - Optional: add progress bar or subtle background for readability.
9) Export with consistent naming:
   Client_EpXX_Topic_Clip01_9x16.mp4

T+18 to T+22 hours: QA
10) QA checklist (per clip):
   - Captions: correct spelling, no missing words, good line breaks
   - Safe area: no text behind UI regions
   - Audio: not peaking, voice clear, music not overpowering (if used)
   - Hook present in first 1.5 seconds
   - No awkward starts/ends; ends with CTA or clean stop
   - File plays on phone without issues

T+22 to T+24 hours: Delivery
11) Upload exports to Drive/Dropbox folder (06_Delivery) and create a simple “Posting Notes” doc:
   - Clip file name
   - Suggested caption (1–2 lines)
   - Suggested on-screen title (hook)
   - Hashtag set (5–12) (optional)
   - Best platform fit (TikTok/Reels/Shorts)
12) Send delivery email with folder link + what to do next + revision policy.
