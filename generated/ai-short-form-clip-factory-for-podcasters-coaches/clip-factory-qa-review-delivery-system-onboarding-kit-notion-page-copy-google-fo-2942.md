## 2) Quality standards (our QA checklist)
We run a final QC pass on every clip before delivery.

### A) Safe zones + framing
- Subject’s face is **not cropped** (especially forehead/chin)
- No key text or captions sit under UI elements (buttons, like/comment stack)
- Content is centered and readable on phones
- If original is widescreen, crop keeps the speaker as the focal point (no awkward side-cropping)

### B) Captions accuracy + readability
- Captions match the spoken audio (names/terms checked)
- Caption timing is tight (minimal lag)
- Line breaks are natural; no overly long single lines
- Profanity/medical/legal claims handled per your preferences (captured in intake)
- Proper nouns and brand terms spelled correctly

### C) Audio leveling
- Speech is clearly audible on mobile
- No clipping/distortion
- Background noise reduced when possible without making voice robotic
- Loudness is consistent across the batch (no “one clip is much quieter” problem)

### D) Jump cuts + pacing
- Cuts feel intentional (no accidental mid-word chops)
- Dead air removed (unless purposeful for comedic/dramatic beat)
- Clip maintains momentum; no slow ramp without payoff

### E) Hook clarity (first 1–2 seconds)
- The opening line is compelling and understandable
- Visuals support the hook (zoom, on-screen text, pattern interrupt where appropriate)
- The clip earns the viewer’s attention quickly (no long setup)

### F) Export + platform formatting
- Delivered in vertical format optimized for TikTok/Reels/Shorts
- Reasonable bitrate (clear, not huge file size)
- No watermarking unless you request it
- Color not overly crushed; text legible

## 3) Review process (how to request revisions)
We provide a review link (Frame.io / Vimeo / unlisted YouTube) or you can comment directly in a shared doc.

**Preferred method:** timestamped comments.
- Example: “00:07 — change caption to ‘…’”
- Example: “00:12 — remove ‘um’ and tighten pause”

### Revision window
- **24 hours** from delivery to submit revision notes for included revisions.
- After 24 hours, we can still revise—just may be scheduled into the next queue.

### What makes review fast
- Consolidate notes per clip (avoid sending 10 separate emails)
- Use timestamps and be explicit about the desired change

---

## 4) What counts as a free revision vs. a paid/scope change
We want you happy, and we also need clear boundaries to protect turnaround times.

### Free revisions (included)
These are fixes to match the agreed spec or correct mistakes:
- Caption typos, misheard words, wrong names/terms
- Captions timing adjustments
- Small framing/safe-zone fixes (e.g., face too close to edge)
- Minor audio leveling tweaks
- Minor trimming for pacing (remove a breath/um, shorten a pause)
- Export/settings issues (wrong resolution, glitch, missing file)

### Paid change requests (out of scope)
These are new creative directions or new work beyond the original request:
- Changing the selected clip moments (new highlights) after delivery
- Adding new branding packages or major style changes after work starts
- Rewriting the hook strategy across the batch
- Heavy b-roll sourcing beyond cueing (unless agreed)
- Multiple rounds of iterative creative direction after the included revision

**Rule of thumb:** If it’s a correction, it’s included. If it’s a new direction, it’s a change request.

# Internal Use — Editor SOP (Consistency + Speed)

## 1) File naming convention
Use consistent naming to prevent lost clips and wrong uploads.

**ClientFolder/**
- **01_Source/** (original long-form file or link notes)
- **02_Exports_MP4/**
- **03_Captions_SRT/**
- **04_Cue_Sheets/**
- **05_Posting_Suggestions/**
- **06_Review_Link_Info/**

**Clip file name format:**
- `CLIENT_YYYY-MM-DD_EP01_CLIP01_HOOK-SHORTTITLE.mp4`
- Matching captions: `CLIENT_YYYY-MM-DD_EP01_CLIP01_HOOK-SHORTTITLE.srt`

## 2) QC steps before delivery (must-pass)
1. Watch full clip once with sound: hook, pacing, no weird cuts
2. Captions pass: spelling + timing + safe zone
3. Audio pass: check volume consistency across at least 2 clips in batch
4. Visual pass: face not cropped; no text under UI
5. Export pass: plays on phone; no artifacting

## 3) Packaging checklist
- MP4s present for all clips promised
- SRT file per clip (or per batch, if requested)
- Cue sheet present and matches clip IDs
- Posting suggestions present and mapped to each clip ID
- Review link included (or Drive folder link)
