## Standard DFY Deliverable (per order)
- 5–20 vertical clips (9:16), each 20–45s (unless otherwise requested)
- Burned-in captions (clean, high-contrast)
- Hook suggestions (first 1–2 seconds)
- Optional b-roll cues as notes (e.g., “cut to screenshot,” “overlay headline,” “zoom punch-in”)
- Platform exports: TikTok/Reels/Shorts-safe framing

**Default specs**
- Resolution: 1080x1920
- FPS: match source (usually 30)
- Captions: large, 2 lines max, safe-zone aware
- Audio: normalize to -14 LUFS-ish (practical target: “clear, not peaking”)

## Production Workflow (T+1h → T+20h)
### Step A — Ingest + Transcribe (T+1h → T+3h)
- Download source / ensure playable
- Quick skim for audio quality and structure
- Generate rough transcript (tool of choice; manual acceptable)

Output:
- A timestamped transcript doc (even rough) stored in `/02_Working`

### Step B — Highlights + Clip Map (T+3h → T+6h)
Find 8–25 candidate moments depending on package size.
Selection rules:
- Starts with tension, contrarian point, or strong promise
- Contains a complete idea within 20–45 seconds
- Avoids long windups; cut into the “meat” fast

Create `ClipMap` (simple table):
- Clip #
- Start–End timestamp
- Working title
- Hook line (first on-screen text suggestion)
- Notes: b-roll cue / emphasis words

### Step C — Edit + Captions (T+6h → T+18h)
For each selected segment:
1) Assemble 9:16 framing (center subject; avoid cropping mouth)
2) Add punch-ins every 2–4 seconds (light motion)
3) Remove dead air; keep cadence tight
4) Captions: large, high contrast; highlight 1–3 keywords max
5) Add optional top header (short, non-clickbait promise)

Caption guidance:
- Keep 2 lines max
- Avoid covering face (safe zone)
- Prefer sentence-case for readability

### Step D — Internal QC (T+18h → T+20h)
Use QA checklist below. Fix issues before client sees.

### Step E — Client Direction Check (optional but recommended)
Deliver 2 “direction check” clips first if this is the first order.
Ask 3 fast questions:
1) Caption style OK? (too dense vs perfect)
2) Hook style OK? (more aggressive vs more natural)
3) Pace OK? (faster cuts vs calmer)

Lock style and finish remaining clips.

## QA Checklist (must pass before delivery)
**Video**
- [ ] 9:16 export, 1080x1920
- [ ] Subject framed well; no excessive cropping
- [ ] No distracting jump cuts / accidental glitches
- [ ] Visual rhythm: punch-ins or pattern interrupts where appropriate

**Captions**
- [ ] Accurate enough (no “reputation-killer” errors)
- [ ] Readable on mobile (size/contrast)
- [ ] Safe zones respected (not blocked by UI)
- [ ] Keywords emphasized sparingly

**Hook + structure**
- [ ] First 1–2 seconds has a clear hook
- [ ] Clip has one core idea; ends cleanly
- [ ] No long preamble

**Audio**
- [ ] Clear voice, no clipping
- [ ] Background noise not overpowering

**Compliance / brand**
- [ ] No disallowed claims (esp. health/finance) unless client approves
- [ ] No copyrighted music unless client provided rights

## Scope control (prevents churn + protects turnaround)
Included:
- Cutting, pacing, captions, minor color/levels, punch-ins

Not included unless explicitly added:
- Heavy motion graphics
- Full re-edits of the original long-form
- Sourcing licensed b-roll libraries

Revision policy (simple):
- 1 revision pass per clip for caption text + timing + hook text
- Additional revisions logged and batch-applied across clips
