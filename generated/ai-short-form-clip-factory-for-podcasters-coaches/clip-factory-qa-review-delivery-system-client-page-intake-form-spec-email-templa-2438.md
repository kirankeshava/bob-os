## 1) What you’re getting (the Delivery Pack)
For each clip, we deliver a **Delivery Pack** containing:
- **Final video (MP4)** — vertical-ready export
- **Captions file (SRT)** — caption text + timing for editing or platform upload
- **B-roll / on-screen cue sheet (PDF or Google Doc)** — where to insert b-roll, screenshots, overlays, emojis, on-screen emphasis, cutaways
- **Posting suggestions (TXT or Doc)** — recommended first-line caption (hook), CTA line, and hashtag set tuned per platform

You’ll receive either a **Google Drive folder link** or a **review link + download link** depending on your preference.

## 2) Our QA standards (what we check before you see it)
We run a final QA pass on every clip to protect quality and your brand.

### A) Safe zones + framing (no cropped faces)
- Subject’s face is **not cut off** by top/bottom UI bars.
- Headroom is appropriate (no “floating chin” or cropped hairline).
- If the source video is wide, the vertical crop stays on the active speaker.
- Any text overlays stay inside safe zones for TikTok/Reels/Shorts.

### B) Captions accuracy + readability
- Captions match spoken words with **no meaning-changing errors**.
- Punctuation and casing are consistent; proper nouns corrected when known.
- No captions cover the mouth/face unnecessarily.
- Caption timing avoids lag (no multi-second drift).

### C) Audio leveling + clarity
- Dialogue is the priority (no music overpowering voice).
- No harsh clipping/distortion; minimize hum/hiss when possible.
- Loudness is consistent across the clip; no sudden jumps.

### D) Hook clarity + pacing
- First 1–2 seconds contain a strong hook (text + visual or punchy cut-in).
- Remove dead air; tight pacing while preserving meaning.
- Jump cuts feel intentional (no jarring mid-word chops).

### E) Export settings (platform compatible)
- Vertical format for short-form: **1080×1920** preferred.
- File type: **.mp4 (H.264)**.
- Frame rate: **30fps** preferred (or source-matched when necessary).
- Audio: **AAC**.

## 3) Review flow (fast, clear, timestamped)
To avoid long back-and-forth, we use timestamp comments.

### Preferred review options (free + easy)
- **Frame.io** (if available) — best for timestamp comments
- **Unlisted YouTube** — comment timestamps (e.g., 0:13 “caption typo”)
- **Vimeo** (unlisted) — timestamp notes if enabled
- If none of the above: we’ll use a **Google Drive** preview + a shared doc for timestamps.

### How to leave feedback (required format)
Please send revision notes as:
- **Clip name** + **timestamp** + **what to change**
- Example: “Clip_03 — 0:07: change caption to ‘pipeline’ not ‘pipelines’.”

## 4) Timelines + revision window
- Delivery includes a first-pass set of clips + review link.
- **Revision window:** **24 hours** from delivery. This keeps production moving and protects turnaround.
- If feedback arrives after 24 hours, we’ll treat it as a new request batch (handled as availability allows).

## 5) What counts as a free revision vs. a change request
We want you happy, but we also need clear boundaries.

### Free revisions (included)
- Caption typos, punctuation, minor timing tweaks
- Small audio balance fixes (voice too quiet, music too loud)
- Minor cropping/safe zone adjustments
- Replacing a b-roll suggestion with another comparable option
- Small on-screen text adjustments that don’t change the concept

### Change requests (not included; scheduled as a new scope)
- Choosing a **different clip moment** from the source after we already edited
- Major rewriting of hooks/angle (“make it about X instead of Y”)
- New versions for additional platforms beyond the agreed formats
- Adding new branded motion graphics packages beyond simple text style
- Significant length changes requiring re-cutting the full piece

If something is ambiguous, we’ll respond with: **(a)** what we can do free, **(b)** what becomes a change request, and **(c)** the fastest path to get you posted.

# Internal: Final QC Checklist (Editor/QA)
Use this before generating the delivery link.

## 1) Project + file hygiene
- Folder name: `ClientName_Project_YYYY-MM-DD`
- Clips named: `Client_Clip01_Topic_1080x1920.mp4`
- Captions named: `Client_Clip01_Topic.srt`

## 2) Visual QA
- No cropped faces (top/bottom safe zones)
- Text inside safe zones
- No accidental black frames
- No weird zoom jitter; stabilize if needed
- Subtitles not covering key facial features

## 3) Captions QA
- Spot-check against audio for meaning errors
- Fix common mistranscriptions (names, brands, terms)
- Caption timing not drifting

## 4) Audio QA
- No clipping
- Dialogue intelligible on phone speaker
- Music bed (if used) is subtle
- Consistent loudness throughout

## 5) Content QA
- Hook lands in first 1–2 seconds
- No dead air; pacing tight
- Cut points are clean (not mid-syllable)
- CTA present if requested

## 6) Export QA
- 1080×1920, H.264 MP4, AAC audio
- File plays on mobile
- Reasonable file size (not excessively large)
