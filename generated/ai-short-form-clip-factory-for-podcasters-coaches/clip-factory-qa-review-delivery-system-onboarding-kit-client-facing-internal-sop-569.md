## 2) Quality standards (QA checklist summary)
We run QA on every clip before delivery.

### A) Framing / safe zones
- Subject’s face is never cropped (forehead/chin/side crop avoided).
- Text stays within safe zones (no UI overlap on TikTok/Reels).
- Headroom is consistent; no distracting zooms or accidental reframes.

### B) Captions accuracy & readability
- Captions match spoken words (names, numbers, brands checked).
- Punctuation/line breaks improve clarity.
- Captions remain readable (contrast, outline/shadow, sizing).

### C) Audio consistency
- Speech is clear and consistent across the clip (no harsh peaks).
- No clipping/distortion; background noise minimized.
- Music (if used) is under voice and not distracting.

### D) Editing clarity
- Hook is obvious in the first 1–2 seconds.
- Jump cuts are clean (no weird mouth frames; no jarring time warps).
- Dead air removed; pacing is “scroll-stopping.”

### E) Export settings
- Vertical 9:16 exports; MP4 H.264.
- No black bars; no watermark unless requested.
- Deliverables tested for playback.

## 3) Review flow (timestamped comments)
To avoid miscommunication, we use timestamped feedback.

### Step 1 — You receive a review link
You’ll get an unlisted review link (Frame.io / Vimeo / unlisted YouTube) **or** a Google Drive preview.

### Step 2 — Leave timestamped notes
Please comment using timestamps (e.g., “00:07 change ‘their’ to ‘there’” or “00:12 tighten the pause”).

### Step 3 — Revision window (24 hours)
You have **24 hours** from delivery of the review link to request revisions. This keeps schedules predictable and ensures we can deliver fast.

## 4) Revisions policy (what’s free vs paid)
We aim to be generous while protecting turnaround times.

### Free revisions (included within the 24h window)
- Caption fixes (typos, mis-heard words).
- Minor timing tweaks (caption sync, slight trim).
- Small framing adjustments if something is cropped or off-safe-zone.
- Audio leveling fixes if something is inconsistent.
- Replacing a b-roll moment if it’s clearly mismatched.

### Paid changes (scope change)
- New creative direction after delivery (“change the whole vibe/style”).
- New hooks/rewrites not based on original request.
- Swapping in a new source file or additional episode content.
- Major re-edit of structure (“turn this into a different angle/story”).
- Additional clip quantity beyond what was purchased.

If you request a scope change, we’ll reply with a fixed add-on price before starting.

# Internal Editor SOP (Ops Only)
## A) File naming
Use this exact convention:
- `CLIENT_EpisodeOrSource_Clip##__HookKeyword__PLATFORM__v1.mp4`
- Example: `AcmeCo_Ep12_Clip03__PricingMistake__IGReels__v1.mp4`

SRT naming must match MP4 base name:
- `...__v1.srt`

## B) Final QC pass (required)
1. **Watch full clip** with sound (no skipping).
2. **Captions check**: names, numbers, jargon; verify at least 3 random timestamps.
3. **Safe zones**: check top/bottom UI overlaps; ensure face not cropped.
4. **Hook clarity**: first 1–2 seconds must be compelling; no slow ramp.
5. **Audio**: no clipping; voice consistently audible; music under voice.
6. **Export**: MP4 plays on mobile; no black bars; correct orientation.

## C) Packaging
Create Delivery Pack folder structure and include:
- Final MP4s
- Matching SRTs
- B-roll cue sheet (doc)
- Posting suggestions (doc)

## D) Handoff checklist
- Confirm clip count matches order
- Confirm every MP4 has matching SRT
- Confirm filenames are consistent
- Confirm review link permissions set to “anyone with link can view/comment”
