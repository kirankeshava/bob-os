D) INTERNAL RULE — INTAKE GATE (EDITOR/OPS SOP)
Rule name: No-Intake, No-Start Gate
Policy: Do not start clip selection, captioning, or editing until the client intake form is submitted and the source content link(s) are accessible.
Exceptions: Only if the client explicitly waives the intake requirement in writing (email or chat) AND acknowledges that revisions may be limited if requirements were not provided.

Operational steps:
1) Intake check (required): Confirm the following are present before starting:
   - Platform targets
   - Caption style preferences
   - Banned words + must-include terms
   - CTA preference
   - Framing/cropping sensitivity
   - Source content link(s) accessible
2) If incomplete: Reply requesting missing items; pause the job timer.
3) If waived: Save waiver message in the project folder and proceed using “default safe settings” (neutral captions, no profanity, conservative crops, minimal b-roll cues).
4) Intake-to-edit handoff: Paste the intake answers into the job brief header so the editor does not need to re-open the form.

Default safe settings (when unclear): 1080x1920, center-safe framing, clean captions (no profanity), moderate highlight words only, -14 LUFS target, -1 dBTP true peak ceiling, no aggressive zooms, hooks in first 1–2 seconds.

This intake gate is mandatory because it directly reduces refunds caused by wrong tone/captions/CTAs and prevents deadline misses due to late requirement changes.
