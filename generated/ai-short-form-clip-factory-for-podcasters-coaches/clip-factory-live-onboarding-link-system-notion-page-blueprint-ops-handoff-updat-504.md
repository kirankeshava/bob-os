### 1) What you’ll receive (Delivery Pack)
For every batch/order, we deliver a folder containing:
1) **Final MP4 clips (vertical)** — ready to post
2) **Caption files (.SRT)** — one per clip
3) **B-Roll / Visual Cue Sheet** — timecoded suggestions for overlays, cutaways, and emphasis moments
4) **Posting Suggestions** — per clip: first-line caption, CTA suggestion, and hashtag set (platform-specific)

---
### 2) Quality standards (our QA checklist)
We run QA on every clip before it’s sent:
- **Safe zones & framing:** faces not cropped; subtitles and key text remain inside platform safe zones
- **Caption accuracy:** correct wording, punctuation, and speaker attribution; no “hallucinated” phrases
- **Readability:** strong contrast, correct line breaks, no overly long lines
- **Audio leveling:** consistent loudness across clips; no harsh peaks; voice is prioritized
- **Jump cuts:** intentional cuts only; no awkward mid-word cuts; pacing matches hook style
- **Hook clarity:** first 1–2 seconds clearly communicate topic/tension; no slow lead-ins
- **Export settings:** correct aspect ratio, bitrate, frame rate; no black bars; no watermark

---
### 3) Review flow (timestamp comments)
**Goal:** fast approvals with minimal back-and-forth.

**We send you a review link** (Frame.io / Vimeo / unlisted YouTube) where you can leave **timestamped comments**.

**How to request revisions:**
- Put feedback directly at the timestamp (example: “00:07 change subtitle wording”)
- Group similar notes (captions, framing, audio) to reduce cycles

**Revision window:** **24 hours** from the time we send the review link.

---
### 4) What counts as a free revision vs paid change (to protect deadlines)
**Free revisions (included):**
- Caption typo fixes, punctuation, minor timing tweaks
- Small audio level adjustments (voice/music balance)
- Minor crop/reframe to fix face cutoffs or safe-zone issues
- Swapping hook text style or removing a single on-screen element
- Fixing anything that violates the QA standards above

**Paid changes (scope changes):**
- Changing the selected moments/quotes (new clip choices)
- Rewriting hooks from scratch across multiple clips
- Adding new branding packages after approval (new fonts/colors/logo rules)
- New deliverables not agreed originally (extra formats, extra clips)
- Additional revision rounds after the 24-hour window (unless we caused the delay)

**If something is our mistake:** we fix it free, fast.

2) **Pre-flight confirmation sent (client email)**

- Ask for missing info (handles, CTA links) before editing starts

3) **Editing production**
- Select moments aligned to hook-first pacing
- Apply captions + styling
- Add b-roll cue notes during editing (timecodes)

4) **Final QA pass (must complete before review link)**
- Safe-zone check on mobile preview
- Caption accuracy spot-check (start/middle/end)
- Audio loudness consistency; remove pops/harsh peaks
- No mid-word jump cuts; hook clarity in first 2 seconds
- Export spec correct (9:16, no bars, correct bitrate)

5) **Create review link**
- Upload watermarked or non-download if desired; enable timestamp comments
- Confirm link permissions (anyone with link can view/comment)

6) **Send review email**
- Include: review link, revision deadline (24h), instructions for timestamp notes

7) **Process revisions**
- Only free revisions within policy; log any scope changes and quote before work
- After revisions, run quick QA again (captions/audio/framing)

8) **Delivery pack assembly**
- Folder structure:
