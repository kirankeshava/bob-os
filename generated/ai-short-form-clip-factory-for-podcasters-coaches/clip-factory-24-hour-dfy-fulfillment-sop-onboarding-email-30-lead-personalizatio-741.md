## 1) 24‑Hour DFY Fulfillment SOP (Payment → Delivery)

### Goal
Deliver a first batch of ready-to-post vertical clips within 24 hours of receiving assets, so we close more deals, earn referrals, and reach the $5k DFY revenue gate before any SaaS build.

### Definition of “Done” (per order)
- Clips exported in 9:16 (1080×1920), H.264 MP4, platform-safe.
- Burned-in captions (readable, high contrast), plus an SRT file.
- Each clip includes: hook-first opening, tight pacing, clean audio, and a title suggestion.
- Delivered via a single folder link with filenames + a simple posting guide.

### Tools (free/low cost preferred)
- Editing: CapCut Desktop / Descript / Premiere (whatever is available).
- Captions: CapCut auto-captions or Whisper (if already available).
- Storage delivery: Google Drive / Dropbox / Frame.io (free tiers).

### Workflow (timestamps assume assets arrive at T0)
**T0–T+15 min: Intake + confirmation**
1) Confirm receipt of assets + requirements via email to client.
2) Create a job folder: 
   - /01-Source
   - /02-Clips-Exports
   - /03-Captions-SRT
   - /04-Thumbs-Titles
   - /05-Notes
3) Log job in CRM: client name, package, deadline, source link, platform focus.

**T+15–T+60 min: Content scan + clip map**
4) Skim the long-form video/podcast at 1.5–2x.
5) Identify 8–15 candidate moments (timestamps) using this filter:
   - Clear pain/problem → insight → payoff
   - Strong opinion / contrarian take
   - Step-by-step tactic
   - “Mistake most people make”
6) Select the best moments based on package size (e.g., 3, 5, or 10 clips).
7) Write a one-line “hook angle” for each chosen clip.

**T+60–T+180 min: Assembly edit**
8) Create 9:16 timeline(s). Crop to speaker; keep eyes near upper third.
9) Remove dead air, repeated words, long breaths; tighten transitions.
10) Add subtle zooms/cuts every 1.5–3.5 seconds to maintain retention.
11) If relevant, add b-roll cues as on-screen text only (avoid time sink):
   - Example: “[B-ROLL: chart going up]” or “[SHOW: screenshot of tool]”.

**T+180–T+300 min: Captions + styling**
12) Generate captions; correct proper nouns, key terms, and numbers.
13) Style rules (default template):
   - 2 lines max, 5–9 words per line
   - Highlight 1–3 key words (brand color)
   - Add punctuation for rhythm
14) Ensure safe margins for TikTok/Reels UI (keep captions above bottom UI).

**T+300–T+360 min: Audio polish + QC**
15) Normalize audio and reduce noise lightly.
16) QC checklist:
   - Hook is in first 1–2 seconds
   - Captions are readable on mobile
   - No cut-off words; no awkward jump cuts
   - File plays correctly; correct orientation

**T+360–T+420 min: Export + package**
17) Export MP4 for each clip; export SRT.
18) Create a simple posting guide (Google Doc or text file):
   - Suggested caption copy (1–2 lines)
   - 5–12 hashtag suggestions (optional)
   - Best platform (TikTok/Reels/Shorts)

**T+420–T+480 min: Delivery + revision window**
19) Send delivery email with folder link and what to do next.
20) Include revision policy: 1 revision round for caption typos and minor timing adjustments.

### Revision rules (keep scope tight)
- Included: caption corrections, minor trim adjustments, hook text tweak.
- Not included unless paid add-on: new clip selection, full re-edit style change, heavy b-roll sourcing.
