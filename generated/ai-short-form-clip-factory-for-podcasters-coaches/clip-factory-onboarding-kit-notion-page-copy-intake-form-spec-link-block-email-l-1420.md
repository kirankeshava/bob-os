## 1) What you’ll receive (Delivery Pack)
For each batch of clips, you’ll receive a **Delivery Pack** folder containing:

1. **Final vertical clips (MP4)**
   - Ready-to-post format for TikTok / Reels / Shorts (as specified in your intake)
2. **Caption file (SRT)** for each clip
   - Matches the spoken words as closely as possible; includes timing
3. **B-roll / visual cue sheet (Google Doc or PDF)**
   - Timestamped notes for suggested cutaways, on-screen text moments, pattern interrupts, and optional b-roll prompts
4. **Posting suggestions (Doc)**
   - Platform-specific recommendations per clip: hook line / first caption line, suggested title (if Shorts), hashtags, and CTA ideas

## 2) Our QA standards (what we check before you see anything)
We run QA on every clip before delivery to prevent avoidable revisions and protect your brand.

### A) Safe zones + framing
- Subject’s face is **not cropped** (top of head/eyes/chin safe)
- Key on-screen text stays within **mobile safe zones** (not hidden by platform UI)
- No important elements cut off at edges; consistent headroom
- Avoid distracting zooms; intentional reframing only

### B) Captions (accuracy + readability)
- Captions match the audio **word-for-word where feasible** (minor cleanup allowed for clarity)
- Correct punctuation for comprehension
- No repeated words / broken lines / awkward wraps
- Readable size/contrast; consistent style per project
- Names/brands/technical terms spelled correctly (based on your intake)

### C) Audio quality + leveling
- Voice is clear, noise reduced where possible
- No clipping/peaking; consistent loudness across clips
- Background music (if used) is below voice and never competes
- No abrupt audio pops at cuts

### D) Edits: jump cuts + pacing
- Cuts are intentional; removes dead air and “ums” when appropriate
- No jarring mid-word cuts; no awkward pauses
- Pattern interrupts used sparingly to keep attention without feeling chaotic

### E) Hook clarity (first 1–2 seconds)
- Opening is compelling and clear (strong line, question, or payoff)
- On-screen hook text aligns with what’s being said
- Hook isn’t clickbait relative to the content

### F) Platform formatting + export settings
- Aspect ratio: **9:16 vertical** unless otherwise requested
- Resolution: **1080×1920** (preferred)
- Codec/container: **H.264 MP4**
- Framerate: matches source where possible (typically 24/30/60)
- Bitrate: high enough to avoid macroblocking on text/face

## 3) Review flow (how feedback works)
We use timestamped comments so changes are fast and unambiguous.

### Step 1 — Review link
You’ll receive a review link via one of these (free-friendly):
- Frame.io (if available on free tier)
- Vimeo Review (if available)
- Unlisted YouTube + timestamped notes

### Step 2 — Leave timestamped notes
Please comment with:
- **Clip filename** (or clip number)
- **Timestamp** (mm:ss)
- **Exact change request** (what to change to what)

Example: “Clip_03 — 00:07: Change caption ‘your’ to ‘you’re’.”

### Step 3 — Revision window
You have **24 hours from delivery** to submit revision notes for included revisions (see Section 5). This keeps turnaround fast and prevents projects from stalling.

If we don’t receive notes within 24 hours, the delivery is considered approved by default (you can still request changes later—those are handled as paid changes if out of scope).

## 4) Delivery format + folder structure
We deliver via a shareable folder link (typically Google Drive).

**Folder naming convention**
- `ClientName_ProjectName_YYYY-MM-DD_Delivery01`

**Inside the folder**
- `/01_FINAL_MP4/`
- `/02_SRT_CAPTIONS/`
- `/03_BROLL_CUE_SHEET/`
- `/04_POSTING_SUGGESTIONS/`
- `/05_REVIEW_LINK_AND_NOTES/` (optional: includes the review URL + instructions)

**File naming convention (clips)**
- `ClientName_Series_Topic_Clip01_15s_v1.mp4`
- Matching SRT: `ClientName_Series_Topic_Clip01_15s_v1.srt`

## 5) Revisions & issue triage (what’s free vs paid)
The goal is fairness: we fix mistakes fast, and we protect both sides from scope creep.

### Included (free) revisions
Free revisions cover anything that is objectively an error relative to the agreed scope/intake:
- Caption typos or misheard words (reasonable corrections)
- Cropped faces / bad framing / safe-zone issues
- Audio leveling issues (too loud/quiet, clipping, distracting noise)
- Export issues (wrong aspect ratio/resolution, corrupted file)
- Missed requested handle/CTA/brand term that was provided in intake
- Simple timing tweaks when the intent is already correct

### Paid changes (out-of-scope)
Handled as a paid change because it alters creative direction or adds new work:
- New hook angle after delivery (“change the whole opening”) 
- Recutting to a different topic/segment not in the chosen clip
- Major style change (new caption design, new font, different pacing style)
- Adding new assets not provided (logos, new b-roll libraries, new music direction)
- Extra versions for A/B testing beyond included deliverables
- Requests submitted after the 24-hour window

### Ambiguous cases (how we decide)
If a request could be either a fix or a change, we default to:
1) Fix if it’s clearly a mistake against the intake/spec.
2) Paid change if it requires re-editing structure, reselecting content, or creating new variations.

If you’re unsure, email us at **agent_bob_replit+clip-factory@agentmail.to** and we’ll confirm before doing extra work.

# Pre-Flight Checklist Mapping (Internal)

- CTA + links provided → verify spelling + placement rules
- Banned words/topics → flag to editor; avoid on-screen text and captions
