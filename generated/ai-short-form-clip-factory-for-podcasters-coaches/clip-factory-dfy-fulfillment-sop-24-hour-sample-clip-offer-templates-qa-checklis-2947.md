## 1) Offers (what we deliver)
### A) 24-hour Sample Clip (foot-in-the-door)
- **Input:** 1 long-form link (YouTube, Spotify video, Zoom, Drive, etc.) + brand notes.
- **Output (within 24 hours of receiving assets):**
  - 1 vertical 9:16 clip (20–45s)
  - Burned-in captions (accurate, readable)
  - Hook options (2–3) in the first 1–2 seconds
  - Title + description copy + hashtags (optional)
  - Export formats: 1080×1920 MP4 + subtitle file (SRT) if requested
- **Goal:** Prove quality and speed; upsell to batch package.

### B) Batch Delivery (72-hour turnaround after approvals)
- **Input:** 1–3 long videos (or one episode) + goals.
- **Output:** 8–20 clips depending on package; consistent template + captions; platform-ready exports.

## 2) Standard Fulfillment Workflow (operator SOP)
### Step 0 — Create a job record (5 min)
Log in CRM/Sheet:
- Client name, link(s), niche, primary CTA, target platforms, deadline
- Brand kit status (fonts/colors/logo), speaker names, preferred hook style
- Package type (Sample / Batch) and clip count

### Step 1 — Intake (send immediately)
Send the **Asset Request** (template below). Do not start work until minimum required assets are received:
- Long-form link/file
- Target platform(s) (TikTok / Reels / Shorts)
- Brand preference (clean vs bold captions), examples they like
- CTA + destination (newsletter, call booking, lead magnet)

### Step 2 — Source selection (15–25 min)
Pick 1–3 candidate moments:
- Strong claim, contrarian take, clear lesson, story pivot, objection handling
- Avoid long rambling setup; look for “payoff” moments

### Step 3 — Clip blueprint (10 min)
For each clip:
- Define **Hook (0–2s)**, **Payoff (2–20s)**, **CTA (last 2–3s)**
- Note any **b-roll cues** (optional): screenshots, headlines, emojis, stock categories

### Step 4 — Edit & captions (45–90 min per clip)
- Set 9:16 crop with smart framing on speaker
- Add captions: 2 lines max, high contrast, punctuation for readability
- Emphasize 1–2 keywords per sentence (bold/color) if style allows
- Add subtle zoom cuts or jump cuts for pacing (every 1.5–3s when needed)

### Step 5 — QA pass (10 min)
Use QA checklist (Section 5). Fix before sending.

### Step 6 — Client delivery (5 min)
Send:
- MP4 download link
- Optional SRT
- Hook options + post copy suggestions
Ask for **one round of feedback** within 24 hours.

### Step 7 — Upsell to batch (same thread)
If Sample Clip delivered:
- Ask: “Want 8–20 clips/month like this?”
- Offer a simple next step: share next episode link + posting cadence.

## 5) QA Checklist (must pass before delivery)
**Captions**
- Names/terms spelled correctly; no obvious mis-transcribes
- 2 lines max; readable on mobile; high contrast
- Punctuation improves comprehension; no walls of text

**Hook & pacing**
- Hook lands within first 1–2 seconds
- Dead air removed; minimal filler
- Visual changes every ~1.5–3 seconds if needed (cuts/zoom/b-roll)

**Framing & export**
- True 9:16 (1080×1920), speaker centered, no cut-off head
- Safe margins for platform UI (top/bottom space)
- Audio normalized and not clipping

**Delivery**
- File plays on mobile
- Link permission set to “anyone with link can view/download”
- Included: MP4 + (optional) SRT + suggested post copy/hooks

## 6) Feedback handling rules (to protect time)
- 1 revision round included for Sample Clip (caption fixes + 1 style tweak)
- For batch packages: collect style notes after first 2 clips, then lock template for the rest
- If client requests a brand-new segment selection, treat as a new clip
