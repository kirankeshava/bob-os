### The goal
We deliver **10 ready-to-post vertical clips in 48 hours**. The only way we can keep that speed is if feedback is **specific and timestamped**.

### What to send (the only format we can act on quickly)
Reply with:
1) **Clip filename** (example: `CLIP_03_Hook_Burn_BellyFat_00-12`)  
2) **Timestamp(s)** (example: `00:03–00:08`)  
3) **Change type** (pick one):
   - **Caption text** (wording, spelling, emphasis)
   - **Hook** (first line, first 1.5s)
   - **Cut/pace** (remove a pause, tighten a sentence)
   - **B-roll cue** (what to show instead)
   - **Branding** (colors, logo, end card)
   - **Compliance** (remove claim, add disclaimer)
4) **Exact instruction** (example: “Replace ‘guaranteed’ with ‘can help’, keep rest the same.”)

### Fast feedback examples (good)
- “**CLIP_02** — `00:00–00:02`: Replace hook text with: ‘Most agents are losing deals for this one reason…’”
- “**CLIP_06** — `00:07–00:10`: Remove the ‘um’ pause; keep captions continuous.”
- “**CLIP_01** — `00:12`: Add b-roll cue: ‘Overlay screenshot of CRM pipeline’.”

### Slow feedback examples (not actionable)
- “Make it more engaging.” (What exactly should change?)
- “Captions feel off.” (Which clip and where?)
- “Can we try a different vibe?” (Which preset, which elements?)

### Timing (protecting the SLA)
- **Feedback window:** Please send notes **within 12 hours** of receiving the review link.
- **No feedback = auto-approval:** If we don’t hear back within the window, we’ll finalize based on the agreed preset and deliver on time.

### What counts as a revision vs a new request
**Included revisions (fast):** caption corrections, small trims, hook line swap, b-roll cue swap, end-card text tweak.  
**Not included (new scope):** rebuilding clips from a different episode, new topic selection, replacing the preset style entirely, new music direction across all clips.

### “One-pass” rule (best results)
Please send **one consolidated message** with all notes across all clips rather than multiple scattered replies.

## 3) Internal Rule Addendum (Ops): Review Loop + Auto-Approval + Escalation

### Review loop standard
- **v1 (primary delivery):** 10 clips + thumbnails/covers if included, sent with review instructions and the Feedback Fast one-pager.
- **v1.1 (revision pass):** only for **timestamped, in-scope** changes.
- **final:** exported deliverables + upload-ready files, packaged and delivered.

### Decision gates (to protect 48-hour SLA)
- If client requests changes **without timestamps**, reply once requesting structured notes (use the Revision Request Template).
- If client requests **out-of-scope** changes, offer: (a) keep SLA and ship in-scope fixes, or (b) treat as new order.

### Auto-approval policy
- If **no feedback within 12 hours** of review link delivery, proceed to final export and deliverables.

### Escalation policy
- If feedback window is missed and client later requests changes that threaten the SLA:
  1) Offer **one in-scope revision pass** on the next business day, OR
  2) Offer a **new batch** (new order) using the same preset.
