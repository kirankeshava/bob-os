Why this matters
Our delivery speed depends on consolidated, timestamped feedback. The fastest clients follow the format below and keep revisions focused on the selected 10 clips (not new clip requests).

Where to leave feedback
1) Frame.io (preferred): comment directly on the timeline.
2) Google Drive: comment on the video (if enabled) or leave notes in a single doc.
3) Email: only if needed—use the exact format below.

Feedback deadline (to protect the SLA)
• Delivery includes a review window. Please send ALL feedback in one consolidated pass within 24 hours of receiving the first cut.
• If we don’t receive feedback by the deadline, clips are considered approved and will be finalized as-delivered to keep the schedule on track.

How to format feedback (copy/paste template)
For each clip, send a single block like this:

CLIP: [Clip ID / Filename]
PRIORITY: Must-fix / Nice-to-have
TIMECODE: 00:00–00:03
CHANGE: [What you want changed]
NOTES: [Any specifics: wording, compliance, tone]

Examples (good)
• CLIP: EP12_C03_ColdEmailHook_v1
  PRIORITY: Must-fix
  TIMECODE: 00:00–00:02
  CHANGE: Replace hook text with “Stop doing this in your cold emails…”
  NOTES: Keep the same pacing; don’t change the rest.

• CLIP: EP12_C07_PriceObjection_v1
  PRIORITY: Nice-to-have
  TIMECODE: 00:08–00:12
  CHANGE: Remove the word “cheap” from captions
  NOTES: Use “lower-cost” instead.

What counts as an in-scope revision (included)
Included revisions are limited to polishing the already-delivered clips:
• Caption corrections (typos, phrasing, punctuation)
• Hook text tweaks (keeping the same clip segment)
• Small trims (tighten start/end by a few seconds)
• Basic layout adjustments (caption position, safe margins)
• Audio leveling tweaks (minor)
• CTA/end-card text changes
• B-roll swap using the same intent (e.g., alternate relevant stock cue)

What does NOT count as an in-scope revision
These requests create new work and break the 48-hour pipeline:
• “Find 5 more clips” / “Swap to a different moment in the episode”
• New scripts, new topics, or new clip selection strategy
• Re-editing the narrative arc (turning one clip into a different clip)
• Full rebrand mid-batch (new fonts/colors/logo after we started)

If you need out-of-scope changes
Reply with the out-of-scope request and we’ll propose options:
• Option A: Treat as an additional batch (new clips)
• Option B: Replace an existing clip 1-for-1 (only if time permits)

Priority levels (use these so we move fast)
• Must-fix: compliance issues, incorrect words, wrong claim, wrong names, major miscaptioning.
• Nice-to-have: style preferences, small pacing tweaks, non-critical b-roll swaps.

Best practices to get the fastest turnaround
• Give one consolidated feedback pass (not 10 separate messages across the day).
• Use timestamps for every change request.
• Don’t mix “new clip ideas” into revision notes—send those as a separate request.
• If your brand has compliance-sensitive terms, list them once and we’ll apply globally.

Copy-ready response snippets (for fast coordination)
Client reminder (24h window):
“Quick reminder: please send consolidated timestamped notes within 24 hours so we can finalize within the 48-hour SLA. Use: Clip ID + Priority + Timecode + Change.”

Scope guardrail:
“Happy to help—note that swapping to a different moment / finding new clips is out of scope for revisions (it’s a new clip request). If you want, we can (A) do an add-on batch or (B) replace one delivered clip 1-for-1 if timing allows.”

Approval confirmation:
“Thanks—implemented the Must-fix notes and applied Nice-to-have where possible. We’re exporting final files now and will deliver in the same folder.”
