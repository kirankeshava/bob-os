## Inputs Required (Client Intake)
Collect via email reply or form copy/paste:
1) Source content link(s): YouTube / Spotify / Drive / Riverside / Zoom (or upload file).
2) Target platform(s): TikTok / Reels / Shorts.
3) Audience + offer: who it’s for + CTA (ebook, call booking, course, newsletter).
4) Brand kit: font (optional), colors (optional), logo (optional), preferred caption style.
5) Tone: punchy / educational / contrarian / calm authority.
6) Do-not-use list: topics, words, claims, competitor mentions.
7) Examples they like: 2–3 links.

If any are missing, proceed with defaults:
- Captions: bold, high-contrast, 2-line max, keyword emphasis.
- Hook style: curiosity + clear outcome in first 1.5s.

## Default Deliverable Specs (Platform-Ready)
Video:
- Format: 9:16 vertical
- Resolution: 1080x1920
- FPS: match source when possible (30fps default)
- Codec: H.264, .mp4
- Loudness target: ~ -14 LUFS integrated (avoid clipping)

Captions:
- Burned-in captions on video (primary)
- Optional: .SRT export (secondary) if requested

Safe zones:
- Keep key text/face centered; avoid bottom ~15% (UI overlays)

File naming convention:
- {Client}_{YYYY-MM-DD}_{SourceTitle}_Clip{01-XX}_{HookKeyword}.mp4
Example: AcmeCo_2026-05-14_PodcastEp42_Clip03_ColdEmail.mp4

## 24-Hour Fulfillment Timeline (Operational Cadence)
### T+0 to T+30 minutes: Intake + Confirmation
1) Create a project folder (see structure below).
2) Log client in CRM: name, email, source link, due time (24h), package, notes.
3) Send confirmation message (template below) and restate delivery time.

### T+30 to T+3 hours: Source Review + Clip Candidate List
1) Skim/scan source at 1.5x–2x speed.
2) Identify 8–15 candidate moments:
   - Strong claim, framework, story pivot, controversy, “how-to” steps, mistake confession.
3) Select the best 3–6 depending on package.
4) For each chosen segment, write:
   - Hook line (0–2s)
   - On-screen headline (3–7 words)
   - B-roll cue notes (optional)
   - CTA end-card text (optional)

### T+3 to T+12 hours: Edit Pass #1 (Assembly)
For each clip:
1) Cut to the tightest start (remove throat-clearing).
2) Add hook reinforcement text in first 1–2 seconds.
3) Add captions (auto + manual cleanup for names/terms).
4) Add jump cuts to remove dead air; pace for retention.
5) Add subtle zooms/crops when helpful.
6) Add b-roll cues as on-screen notes OR insert simple b-roll if available (client-provided).

### T+12 to T+18 hours: Edit Pass #2 (Polish)
1) Caption QC: punctuation, correct words, highlight keywords.
2) Visual QC: safe zones, no covered faces/text, consistent margins.
3) Audio QC: normalize, remove harsh peaks.
4) Add end frame with CTA if requested (e.g., “Comment ‘CLIPS’ for the framework”).

### T+18 to T+22 hours: Export + Packaging
1) Export platform-ready mp4.
2) Optional exports:
   - .SRT captions
   - Hook/Caption copy in a text doc
3) Upload to delivery folder and generate share link.

### T+22 to T+24 hours: Delivery + Revision Handling
1) Send delivery message with link and posting suggestions.
2) Offer one revision round (tight scope) within 48 hours:
   - Allowed: caption fixes, start trim, hook text swap, CTA swap, small pacing tweaks
   - Not included: new clip selection, full re-edit, heavy b-roll sourcing unless add-on

## QA Checklist (Must Pass Before Delivery)
Content:
- Hook appears within first 1.5 seconds
- No fluff at start; clip starts mid-sentence if needed
- Clip length matches goal (15–45s typical; up to 60–90s if high-retention)

Captions:
- No obvious mis-transcribes for names/brands
- Keyword emphasis used sparingly (not every word)
- Captions do not cover face or UI

Visual:
- 9:16 is true vertical (no black bars unless intentional)
- Subject is centered; safe zones respected

Audio:
- No clipping, harsh sibilance minimized
- Volume consistent across clips

Packaging:
- Filenames correct
- Share link works (viewer permissions)
