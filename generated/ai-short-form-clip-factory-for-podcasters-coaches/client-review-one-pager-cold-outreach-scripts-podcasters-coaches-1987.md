CLIENT-FACING ONE-PAGER
How to Review Your Clips + Request Revisions (DFY Clip Factory)

Where your clips are delivered
You’ll receive either (a) a Google Drive folder link or (b) a Frame.io / review-link style page, plus a simple index list of clips.

What to do first (takes ~10–15 minutes)
1) Watch all clips once WITHOUT pausing.
2) Re-watch only the 2–3 clips you’re least sure about.
3) Approve the winners quickly so you can start posting while we refine anything that needs changes.

How to leave feedback (required format)
To request changes, please reply with ONE message using this format per clip:
- Clip File Name: (example) CLIP_03_Hook-Offer_TikTok_v1.mp4
- Timestamp(s): 00:03–00:07
- Change Requested: (one sentence)
- Goal: (ex: clearer hook / remove filler / stronger CTA / add emphasis)
- Must-keep words (optional): (ex: “10k/month”)

Examples of “good” feedback
- “CLIP_02… 00:00–00:02: replace opening caption with ‘Stop doing this on LinkedIn…’ to make the hook more direct.”
- “CLIP_07… 00:12–00:16: cut the pause and tighten pacing; keep the meaning the same.”

What counts as a revision (included)
Included revisions focus on improving what’s already there:
- Caption wording tweaks, emphasis, timing
- Minor trims for pacing (remove pauses/filler)
- Hook swap using nearby wording from the same segment
- CTA end-card text adjustments

What does NOT count as a revision (would be a new request)
- Changing the clip to a different topic/segment not in the chosen timestamp range
- New b-roll sourcing requirements beyond simple cues
- Rebuilding an edit into a completely different style after approvals

Decision deadline (protects the 48-hour workflow)
Please send revisions within 48 hours of delivery. If we don’t hear back, we’ll consider the batch approved so you can keep momentum.

If you want a “batch style lock” (recommended)
Pick your top 2 clips and say: “Use these as the style reference going forward.” That dramatically increases consistency across future batches.
