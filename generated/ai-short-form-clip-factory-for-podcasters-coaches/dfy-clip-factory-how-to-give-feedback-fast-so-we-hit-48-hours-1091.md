How to Give Feedback Fast (So We Hit 48 Hours)

This guide ensures your clips ship on time with minimal back-and-forth. It’s designed for speed and consistency, not perfection.

Official Clip Factory site (for legitimacy + details): https://6ff73b98-dcff-4f53-a10d-9ad392f63342-00-1n4h4nuiq6w8j-dbyyfbxp.picard.replit.dev/sites/3
Support / Contact: agent_bob_replit+clip-factory@agentmail.to

1) Where to leave feedback
You’ll receive either a Frame.io review link or a Google Drive folder (plus a review method).

Preferred: Frame.io (best for timestamped notes)
If Google Drive: comment on the specific file and include timestamps in the comment text.

2) The only format we accept (copy/paste template)
For each requested change, comment like this:

[CLIP ID / Filename]
Timestamp: 00:00–00:03
Request type: (Caption / Hook / Cut / B-roll / On-screen text / End card / Crop)
Change: “Replace ‘X’ with ‘Y’”
Priority: Must-fix / Nice-to-have

Examples:
- CLIP_03 | Timestamp 00:01–00:02 | Hook | Change: “Start at ‘…and that’s why…’ instead” | Must-fix
- CLIP_07 | Timestamp 00:10 | Caption | Change: “Correct spelling: ‘principal’ not ‘principle’” | Must-fix
- CLIP_02 | Timestamp 00:14 | B-roll | Change: “Add quick gym clip here” | Nice-to-have

3) Give ONE consolidated round
To protect the 48-hour SLA, consolidate feedback into a single round.

Do:
- Review all 10 clips first
- Leave all comments
- Send one final message: “Feedback complete”

Don’t:
- Send comments across multiple emails/DMs over 24–48 hours
- Ask for multiple creative directions mid-batch (that becomes a new batch)

4) Decide your top 3 “Must-fix” items
If everything is “urgent,” nothing is. Mark up to 3 items as Must-fix per clip.
Nice-to-have edits get applied only if they don’t jeopardize delivery.

5) Feedback window (protects the 48-hour turnaround)
Your feedback window is 12 hours from delivery of the first proof/review link.

If feedback arrives within 12 hours: we will deliver revisions within the SLA.
If feedback arrives after 12 hours: revisions move to the next available slot (still handled fast, but SLA timing resets).

6) What counts as a revision (and what counts as new scope)
Included revisions (within the defined revision round):
- Fix captions/typos
- Minor trim adjustments (tightening dead air, removing filler)
- Small hook swap within the same segment
- B-roll cue swaps (minor)
- End-card CTA text tweaks

New scope (treated as a new request / new batch):
- Changing the style preset (fonts/colors/layout) after clips are built
- Asking for entirely new clip selections (“pick 10 different moments”)
- Major narrative restructure per clip
- Rebuilding the batch for a different platform strategy

7) Fast approvals = more output
If you approve quickly, we can often:
- Deliver earlier than 48 hours
- Use saved time to polish pacing/pattern interrupts
- Potentially add 1–2 bonus alternates (only when time allows; not guaranteed)

8) If something is wrong (audio, names, claims)
Mark it Must-fix and specify the correction.
If it’s compliance-sensitive (medical/financial claims, client names, legal terms), explicitly state the approved wording.

9) Final approval message (required)
After feedback is left, reply with one of:
- “Approved — ready to post.”
- “Revisions requested — see comments.”
