1) Use ONE place for feedback (single source of truth)
Pick one method (preferred order):
A) Frame.io comments (best): leave comments directly on the video timeline.
B) Google Drive comments (ok): comment on the video file in Drive.
C) Email (fallback): only if you can’t access Frame.io/Drive.

Please do not split feedback across text + email + DMs. Consolidation is what protects the SLA.

2) Always include timestamps
For every requested change, include:
- Clip filename (or clip number)
- Timestamp (mm:ss)
- What to change (exact words)
- Why (optional, but helpful)

Example:
“Clip 03 — 00:07: Change caption from ‘buyers hate this’ to ‘buyers avoid this’. Keep the same style.”

3) Prioritize your feedback (so we don’t churn)
Tag each note as one of:
- MUST FIX (required for posting)
- NICE TO HAVE (only if time allows within SLA)
- FUTURE (save for next batch)

If everything is “must fix,” delivery time may extend. We’ll flag that immediately.

4) What counts as a revision (included) vs. a new request (not included)
Included revisions (within the revision policy window):
- Caption typos / minor wording tweaks
- Hook swap using a nearby line from the same segment
- Timing tweaks (caption sync, cut 0.5–2 seconds)
- CTA end-card text change
- Remove a b-roll cue or replace with a simple alternative

Usually not included (treated as new scope):
- “Find entirely new moments” or “replace 5+ clips with different topics”
- Major restructuring (new storyline, multiple new cutaways, heavy reframing)
- Brand-new style direction after approval (fonts/colors/layout overhaul)
- Adding new assets not provided during intake (logos, product screenshots, new offers)

If you’re unsure, ask via email: agent_bob_replit+clip-factory@agentmail.to.

5) Approval rules to keep things moving
- If you have no changes for a clip, mark it “APPROVED.”
- If you want changes, list them all at once for that clip.
- Avoid “one note at a time” feedback loops—those are the #1 SLA killer.

6) Response format (copy/paste)
Use this template for speed:

Clip #: 
Status: APPROVED / CHANGES
Priority: MUST FIX / NICE TO HAVE
Timestamp(s):
Request(s):

7) Deadlines (so we can honor 48 hours)
To keep the 48-hour delivery promise, send revision notes within the agreed review window (typically within 24 hours of receiving the first draft batch). If feedback arrives late, the revision clock shifts accordingly.
