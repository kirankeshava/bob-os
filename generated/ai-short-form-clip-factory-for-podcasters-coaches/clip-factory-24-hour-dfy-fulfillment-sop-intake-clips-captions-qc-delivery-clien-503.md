Deliver ready-to-post vertical short-form clips from a long-form podcast/video within 24 hours of receiving assets, with consistent style and minimal revisions. This SOP is designed to protect margin while we close DFY packages and reach the $5k revenue gate before building SaaS.

1) N vertical clips (9:16) in MP4 format, 1080x1920, <60s each unless specified.
2) Burned-in captions (readable on mobile, within safe zones).
3) A “Posting Pack” doc: hook text options, suggested titles, and recommended posting order.
4) File organization: clearly named files + a single shared folder link.
5) A delivery email sent from agent_bob_replit+clip-factory@agentmail.to that includes the website URL above.

PHASE 2 — SOURCE ACQUISITION + PREP (10–20 MIN)
1) Download or import the long-form.
2) Quick skim for audio quality and pacing.
3) Note any immediate constraints:
- Heavy crosstalk
- Very quiet audio
- Screen-share heavy (may need reframing)
- Music copyrighted (avoid if possible)

Optional quick audio cleanup:
- Normalize loudness to comfortable speech level.
- Reduce noise lightly (don’t artifact).

PHASE 3 — FIND CLIP CANDIDATES (30–60 MIN)
Method A (fastest): “Moment scan”
- Scrub at 1.5–2x.
- Mark timecodes for:
  a) Contrarian takes
  b) Step-by-step frameworks
  c) Punchy one-liners
  d) Mistakes + fixes
  e) Strong emotional moments

Clip selection rules:
- Start as close as possible to the hook (no long wind-up).
- Target 20–45s (unless client asked otherwise).
- One idea per clip.
- Avoid inside jokes that don’t land without context.

Hook formulas to look for:
- “Stop doing X. Do Y instead.”
- “If you’re [persona], you need to hear this.”
- “I wasted [time/money] until I learned this.”
- “Most people get X wrong…”

Deliverable target:
- For 5 clips: find 8–10 candidates, then pick the best 5.
- For 10 clips: find 15–18 candidates.

PHASE 4 — EDITING + FORMAT (60–120 MIN)
Project settings:
- Sequence: 1080x1920, 30fps (or match source), 10–20 Mbps export.
- Frame: center on speaker; reframe for 9:16.

Standard template rules (keep consistent):
- Safe zones: keep subtitles and key text away from bottom 15% (UI overlays).
- Speaker framing: eyes around upper third.
- Jump cuts: remove dead air >0.5–0.8s unless intentionally dramatic.
- Add subtle zooms every 5–8 seconds if it improves retention (optional).

B-roll cues (optional, if time allows):
- If client uses b-roll, insert simple placeholders or notes in Posting Pack:
  “B-roll idea: show [concept]”
Don’t over-invest unless sold as an add-on.

PHASE 5 — CAPTIONS (30–60 MIN)
Caption style defaults:
- White text with high-contrast shadow/outline.
- Font: clean sans (CapCut default is fine).
- Size: readable on phone.
- Highlight 1–3 keywords per sentence (optional).
- Avoid 3+ lines on screen.

Caption QA:
- Correct names, brands, numbers, and technical terms.
- Remove filler words only if it doesn’t desync.
- If profanity policy is “bleep,” bleep audio and captions.

PHASE 6 — QC CHECKLIST (10–15 MIN)
Before exporting final:
- Hook within first 1–2 seconds (or a compelling visual/text immediately).
- No long pauses.
- Captions aligned and readable.
- No text in unsafe bottom area.
- Audio not clipping/distorting.
- Clip ends cleanly (no mid-word cut).
- Filename matches convention.

File naming convention:
ClientName_TopicKeyword_Clip01_YYYYMMDD.mp4
Example: AcmeCoach_ColdOutreach_Clip03_20260409.mp4
