## 1) Standard DFY deliverable definition
**One “episode” input** (YouTube link, podcast video file, Zoom recording, or Drive link) becomes:
- 10–30 vertical clips (9:16) depending on package
- Each clip includes:
  1) Hook line (first 1–2 seconds) as on-screen text
  2) Burned-in captions (word-level or phrase-level)
  3) Basic b-roll cues (text notes or simple cutaways if provided)
  4) Platform-safe framing (safe margins for TikTok/Reels)
  5) Optional CTA end card

**Default format**
- 1080x1920 (9:16), 30fps, H.264 .mp4
- Audio normalized for speech clarity (no clipping)
- Captions: high-contrast, large, centered low/mid with safe margins

## 3) Production workflow (timeboxed)
### Step A — Project setup (10 minutes)
Create folder structure:
- /CLIENT_NAME/YY-MM-DD_EPISODE_TITLE/
  - 01_input/
  - 02_transcript/
  - 03_clips_wip/
  - 04_clips_final/
  - 05_thumbnails_titles/
  - 06_notes_delivery/

Naming convention (consistent):
- CLIENT_EpShort_Topic_Clip01_HOOK.mp4
- Example: ACME_E12_SalesCalls_Clip03_StopDoingThis.mp4

### Step B — Transcript + highlights (30–60 minutes)
- Generate transcript (any tool available) and skim for:
  - Clean 15–45 sec moments that stand alone
  - Strong claims, tactical steps, hot takes, lists, “mistakes”, “do this not that”
- Create a shortlist of 2× desired clip count (e.g., for 15 clips, shortlist 30 moments)
- Select final set prioritizing:
  1) Clarity without context
  2) Emotional energy / conviction
  3) Novelty or specificity

### Step C — Clip assembly (2–6 hours depending on volume)
For each chosen segment:
1) Hard cut to start on a strong word (remove dead air)
2) Add hook text overlay within first 1–2 seconds
3) Add captions (clean punctuation, remove filler where possible)
4) Reframe to 9:16 (speaker centered, safe margins)
5) Add simple b-roll cues as notes OR insert b-roll if provided
6) Add subtle progress bar only if it matches brand (optional)
7) End cleanly; if CTA requested, add 1–2 sec end card

### Step D — Quality checklist (per clip, 2 minutes)
- Captions readable on mobile? (contrast + size)
- Hook appears immediately and matches what’s said?
- No awkward jump cuts mid-word
- Audio balanced and not peaking
- No captions covering UI zones (bottom area)

### Step E — Batch export + delivery (30 minutes)
Export finals into /04_clips_final.
Create a single Delivery Note doc in /06_notes_delivery with:
- Clip list (filename + 1-sentence summary)
- Suggested post copy (1 line each optional)
- Any issues/questions
