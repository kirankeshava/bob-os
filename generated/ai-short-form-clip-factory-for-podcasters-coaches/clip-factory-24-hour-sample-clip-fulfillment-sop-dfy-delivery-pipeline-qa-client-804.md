Purpose
Deliver a “sample clip” (or first batch from a DFY package) within 24 hours of receiving the client’s link/assets, with consistent quality and a handoff that feels professional enough to close a retainer. This SOP supports the revenue gate: sell DFY first, learn what clients want, then consider SaaS only after $5k collected.

Service Promise (what the client experiences)
- Turnaround: 24 hours (business days) from receiving a usable link/file + brand notes.
- Deliverable for sample: 1 fully edited 9:16 vertical clip (20–45s) with burned-in captions + hook + CTA (if requested).
- Deliverable for paid batch (typical): 3–10 clips depending on package, delivered in a shared folder with clear filenames.

Inputs Required (Intake Checklist)
Client must provide:
1) Source content link(s): YouTube, Riverside, Drive link, or uploaded file.
2) Target platforms: TikTok / Reels / Shorts (choose 1–3).
3) Branding: logo (optional), brand colors (hex if possible), font preference (optional).
4) Caption style preference: clean / bold / karaoke / word-by-word emphasis.
5) Hook style: contrarian, curiosity, list, myth-busting, story.
6) CTA + destination: “Follow for X”, “Link in bio”, “Book a call”, etc.
7) Any forbidden topics/words.

Production Pipeline (operator steps)
Step 0 — Job Setup (5 minutes)
- Create a project folder: /ClientName/YYYY-MM-DD_SourceTitle/
- Create subfolders: /01_Source, /02_Transcripts, /03_Selects, /04_Exports, /05_Thumbnails (optional)
- Save the client intake notes in a text doc: ClientName_intake.txt

Step 1 — Source Retrieval (10–20 minutes)
- Download the source video/audio (highest available quality).
- Confirm audio quality: no severe clipping; voices intelligible.
- If unusable, request alternate link immediately (do not proceed).

Step 2 — Transcript & Selects (20–40 minutes)
- Generate transcript (manual tool or service; no SaaS build).
- Identify 3–5 candidate moments using these rules:
  - Clear standalone context in <45 seconds.
  - One strong idea: a claim, insight, step-by-step, or emotional moment.
  - Minimal “as I said earlier” references.
  - Prefer moments with stakes: numbers, mistakes, unpopular opinion.
- Pick the best 1 moment for sample (or best N for paid batch).

Step 3 — Edit Assembly (30–60 minutes)
- Create a 9:16 sequence at 1080x1920.
- Frame the speaker for vertical (centered, eyes near upper third). If two speakers, cut or reframe to keep the active speaker prominent.
- Remove dead air >300ms where it improves pace.
- Add a hook:
  - On-screen hook line in first 0–2 seconds (short, high-contrast).
  - Optional: quick jump cut right after hook to maintain retention.
- Captions:
  - Burned-in, high-contrast, safe margins.
  - Highlight 1–3 keywords per sentence (color or bold).
  - Keep lines short (max ~32 characters per line).
- Add light b-roll cues if relevant (optional for sample): 1–2 quick overlays only when they clarify meaning.
- Add subtle background music only if it does not compete with speech (keep very low).

Step 4 — Branding Pass (10–15 minutes)
- Add small logo bug (optional) bottom corner.
- Add consistent caption styling aligned with brand colors (if provided).
- Ensure nothing is blocked by platform UI (TikTok/Reels safe zones).

Step 5 — QA Checklist (5–10 minutes)
Before export/delivery, confirm:
- Hook appears immediately and matches the clip’s actual content.
- Captions are accurate (names/terms correct).
- No clipped audio peaks; voice is clear.
- No jarring mid-word cuts.
- Visual framing is stable and not overly zoomed.
- CTA (if included) is correct and non-spammy.
- Export is 1080x1920, H.264, AAC audio.

Step 6 — Export Settings (2–5 minutes)
- Format: MP4
- Resolution: 1080x1920
- FPS: match source where possible (30fps common)
- Bitrate: high enough for clean text (avoid mushy captions)
- Filename convention:
  ClientName_SourceTitle_Clip01_HookKeyword_9x16.mp4

Step 7 — Delivery (5 minutes)
- Upload to Drive/Dropbox folder.
- Include a short delivery note + next-step upsell prompt.

Turn Delivery Into Revenue (post-delivery script)
After delivery, ask one friction-reducing question:
- “Want the next 5 clips pulled from the same episode, or should I use your most recent one?”
Then present a simple choice:
- “I can do $X for a one-off batch or $Y/mo for weekly clips.”

Time Budget (to protect 24-hour SLA)
- Setup + download: 20m
- Transcript + selects: 40m
- Edit + captions + hook: 60m
- QA + export + delivery: 20m
Total target: ~2h40m per sample clip.

Notes / Non-goals
- Do not build product features or custom tools as part of fulfillment.
- Use the fastest available tools; prioritize speed + consistency.
- Every delivery should collect 1–2 data points: preferred caption style, ideal clip length, and whether they’d pay monthly for consistent output.
