8) 15-MINUTE PILOT ONBOARDING CHECKLIST (speed-to-live)
Collect (5 minutes):
- Business name/location + timezone
- Booking system type (calendar tool / practice management / manual)
- Appointment types + average appointment value (or range)
- Current no-show rate estimate (or last 4 weeks no-shows + total appts)
- Reminder schedule preferences (e.g., 24h + 2h; morning-of only; etc.)
- Reschedule rules (allowed window, same-day yes/no)
- Owner escalation contact (email + phone)

Configure (5 minutes):
- Message templates (friendly, brand name, clear CTA)
- Two-way intents: YES / NO / RESCHEDULE + rule-based overrides
- Fail-safes: if calendar/API fails, alert owner escalation contact
- Opt-out/HELP responses enabled

Verify (5 minutes):
- Send test reminder + confirm YES path updates status
- Test NO/RESCHEDULE path triggers reschedule instructions
- Test STOP suppresses immediately
- Verify timezone by scheduling a test appointment near boundary times

Baseline metrics to capture before go-live:
- Last 4 weeks: total appointments, no-shows, late cancels (if tracked)
- Average appointment value (or average revenue per slot)

Outcome metrics during pilot (weekly rollup):
- # reminders sent, # confirmations, # reschedules, # opt-outs
- Estimated recovered appointments = (baseline no-show rate – pilot no-show rate) * appointments
- Estimated recovered revenue = recovered appointments * avg appointment value
