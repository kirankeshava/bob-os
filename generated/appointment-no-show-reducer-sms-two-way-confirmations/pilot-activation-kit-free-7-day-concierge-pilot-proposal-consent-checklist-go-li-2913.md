## 2) SMS Consent + Opt-In Compliance Checklist (must pass before go-live)

**Owner authorization (required)**
- [ ] Written confirmation (email is fine) that the location authorizes Appointment No-Show Reducer to send SMS reminders to their customers for scheduled appointments.
- [ ] Escalation contact named (owner/manager/front desk lead) + phone/email for urgent issues.

**Patient/client consent basis (confirm which is true)**
- [ ] Customers provide phone number during booking AND are informed they will receive appointment-related texts (transactional/operational).
- [ ] If business has explicit SMS consent checkbox/terms, capture a screenshot or wording.
- [ ] Confirm message category is appointment-related (no marketing).

**STOP/HELP + opt-out handling**
- [ ] Any reply containing STOP/UNSUBSCRIBE/CANCEL/END/QUIT => immediate opt-out.
- [ ] HELP => send help message with support email: agent_bob_replit+no-show-bot@agentmail.to.
- [ ] Opt-out is persistent per phone number; no further messages unless re-opt-in is documented.

**Quiet hours + timing**
- [ ] Location timezone confirmed.
- [ ] Quiet hours policy defined (e.g., no texts before 8am or after 8pm local).
- [ ] DST behavior confirmed (texts send at correct local time even across DST).

**Content safety**
- [ ] Messages contain business/location name and clear purpose (“appointment reminder”).
- [ ] No sensitive details included (no medical procedure names, etc., unless client explicitly requests and it’s allowed).

**Failure alerting**
- [ ] If calendar/booking integration fails OR reminders cannot be sent, escalation contact is alerted within same business day.

## 5) QA-to-Outcome Mapping (so defects tie to value)

- **Timezone/DST wrong** → reminders late/early → lower confirmations + higher no-show risk → shows up as drop in confirmation rate and incident count.
- **Calendar write-back fails** (reschedule not saved) → double-booking / missed appointment → must trigger same-day escalation alert.
- **Reply parsing misclassifies** (e.g., “Yes but running late”) → wrong action → track misclassifications and add rule override keywords.
- **STOP not honored** → compliance risk → immediate rollback + verify suppression list.
- **Threading breaks** (new message loses context) → more manual work + customer confusion → measure manual interventions/day.

This ensures weekly reporting reflects both wins (recovered revenue) and reliability (incidents avoided/resolved).
