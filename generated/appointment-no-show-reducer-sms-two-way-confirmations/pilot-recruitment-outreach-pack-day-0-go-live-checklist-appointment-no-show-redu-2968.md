DAY-0 to DAY-2 GO-LIVE CHECKLIST (for concierge pilot)
========================
Goal: Convert interest → live messaging with measurable baseline and safe fail-safes.

Day-0 (during/after the 15-min call)
1) Confirm decision-maker + escalation contact (name + phone + email).
2) Confirm timezone + business hours (including lunch/closed days).
3) Capture baseline metrics (last ~4 weeks):
   - Approx appointments/week
   - No-shows/week (or %)
   - Avg appointment value ($)
   - Current reminder process (none / manual / tool)
4) Consent/opt-in confirmation:
   - “You confirm you have consent to message clients about their appointments and will include opt-out instructions.”
5) Reminder policy:
   - Default suggestion: 24h reminder + 2h reminder
   - Confirm language tone (formal vs friendly)
6) Reschedule rules:
   - Allowed window (e.g., >4 hours before appointment)
   - Where to route reschedules (link, phone, or concierge)
7) Waitlist rules (if used):
   - Who is eligible, how far out, max offers per slot
8) Choose one “test appointment” to run end-to-end before enabling for all.

Day-1 (go-live)
1) Run test appointment end-to-end:
   - Send reminder → reply YES/NO/RESCHEDULE → correct threading → correct update/notification
2) Verify STOP/HELP behavior (opt-out immediately, log it, no further messages).
3) Confirm owner alerts path if anything fails (calendar/API failure, unknown reply, double-book risk).

Day-2 (stabilize)
1) Review logs for unknown intents or misclassifications.
2) Add rule overrides for any recurring keywords (e.g., “running late”, “can’t make it”, “cancel”).
3) Confirm measurement events are captured: sent, delivered, reply, confirmed, rescheduled, slot filled.

This checklist ensures pilots go live quickly while protecting customer trust (consent/opt-out) and ensuring we can produce a week-1 value report with baseline vs. pilot deltas.
