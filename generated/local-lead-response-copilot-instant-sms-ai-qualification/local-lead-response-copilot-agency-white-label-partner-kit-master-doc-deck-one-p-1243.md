## D) Implementation SOP (Agency → Client → Launch)

### Phase 0 — Pre‑sale qualification (agency)
Confirm the client has:
- Consistent lead flow (forms/ads)
- Clear service area and job types
- Someone to answer booked calls/appointments
- Permission to text leads (TCPA-compliant consent on forms/ads)

### Phase 1 — Client Intake (Day 0)
Collect:
1) Business name, location(s), timezone
2) Services offered + service radius
3) Disqualifiers (no-go jobs, minimum price, etc.)
4) Booking preference (Calendly link / call routing)
5) Lead sources (form URL, FB lead form, webhook, Zapier)
6) Brand voice (friendly/professional) + any compliance notes

### Phase 2 — Build (Day 1)
- Connect lead source → Copilot trigger
- Configure instant SMS copy (first message)
- Configure AI qualification questions (3–6)
- Configure routing:
  - If qualified → booking link / notify office
  - If unqualified → polite close + optional nurture
  - If no response → follow-up sequence

### Phase 3 — QA (Day 2)
Test with 5–10 dummy leads:
- Response time logging
- Correct qualification outcomes
- Booking/call routing works
- Confirm opt-out language (e.g., “Reply STOP to opt out”)

### Phase 4 — Launch (Day 2–3)
- Turn on live lead routing
- Monitor first 20 conversations
- Adjust questions to reduce drop-off

### Phase 5 — Optimize (Week 2 onward)
- Identify bottlenecks:
  - Low contact rate → tweak first message + timing
  - Low qualified rate → adjust criteria or messaging expectations
  - Low booked rate → change CTA, booking friction, office handoff

### Ongoing — Monthly reporting (agency)
Send KPI report monthly (template below). Include:
- KPI summary
- What changed (script tweaks)
- Next month plan
