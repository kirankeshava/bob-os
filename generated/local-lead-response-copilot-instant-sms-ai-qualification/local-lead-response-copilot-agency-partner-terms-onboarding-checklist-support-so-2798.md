# Agency Onboarding Checklist (Partner-Level)

Use this for your first 5 accounts and every new account afterward.

## A) Partner Setup (Once)
1. Confirm your **wholesale tier** (min 3/5/10 accounts).
2. Decide your **retail package names** (e.g., “Instant Lead Response,” “AI Lead Qualifier,” “Booked Calls System”).
3. Create your internal pipeline stages (New Lead → Contacted → Qualified → Booked → Not Qualified → No Response).
4. Decide default qualification questions by niche:
   - Service needed
   - Zip code / service area
   - Timeline
   - Budget range (optional)
   - Property type (optional)
5. Decide routing rules:
   - Booked → notify owner + calendar
   - Qualified-not-booked → send to CSR for manual booking
   - Not qualified → polite close
   - No response → 2–4 follow-ups over 24–48 hours

## B) Per-Client Intake (15 minutes)
Collect these in one email/form:
- Business name + hours
- Primary contact + best number for notifications
- Service(s) offered + service area
- Lead sources (FB lead form / website form / landing page / CRM)
- Booking link (Calendly/Housecall Pro/ServiceTitan, etc.)
- Qualification rules (what counts as qualified)
- Any compliance notes (TCPA language on forms, existing opt-in)

## C) Per-Client Build & QA (Provider Executes)
- Connect lead source → trigger instant SMS
- Configure question flow + branching
- Configure booking/routing notifications
- Test: submit dummy lead → verify SMS, qualification, booking, notifications

## D) Client Launch (Day 1)
- Turn on automation
- Monitor first 20 leads closely
- Adjust 1–2 questions max (avoid overfitting)

## E) 7-Day Optimization
- Review KPI template
- Tune follow-up timing
- If contact rate is low: adjust first message + shorten questions
- If qualified rate is low: re-check lead targeting/offer and qualification thresholds

# Internal Partner Support SOP (Provider-Side)

## 1) Triage Categories
**Category 1 — Outage/critical:** no messages sending, booking link broken across accounts, provider errors.
- Target response: Tier A 4 hours / Tier B 8 hours / Tier C 1 business day.

**Category 2 — Deliverability:** messages sent but not received, high “no response” rates suspicious, carrier filtering.
- Action: verify first message content, remove spammy words, confirm local area code if possible, reduce link usage in first message.

**Category 3 — Configuration:** wrong routing, wrong questions, wrong hours, wrong booking link.
- Action: confirm intake data, apply patch, retest with dummy lead.

**Category 4 — Reporting/strategy:** interpretation of KPIs, optimization ideas.
- Action: provide 1–3 changes max; set next review date.

## 2) Standard Debug Checklist
1. Confirm lead source trigger fires (timestamp).
2. Confirm SMS send event recorded.
3. Confirm lead phone format valid.
4. Confirm opt-out status not set.
5. Confirm follow-up schedule enabled.
6. Confirm booking/routing destinations (email/phone/calendar link) correct.

## 3) Opt-Out / Compliance Requests
- If Partner reports a compliance concern, pause messaging for that account until clarified.
- For individual leads requesting removal: mark opted-out and confirm to Partner.

## 4) KPI Review Cadence (Monthly)
Each month, deliver or help Partner produce:
- Median response time
- Contact rate
- Qualified rate
- Booked rate
- Lead status breakdown
And one “Next month focus” recommendation:
- Improve contact (first message + timing)
- Improve qualification (question order)
- Improve booking (calendar friction + availability)

## 5) Escalation Rule
If an issue impacts **3+ accounts** or prevents booking, escalate to “critical,” post an update to Partner within the SLA window, and provide an ETA + workaround.
