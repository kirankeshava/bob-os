SECTION 4 — IMPLEMENTATION SOP (AGENCY DELIVERY)
============================================

Objective:
Launch each client’s Copilot fast (48–72 hours typical) and ensure tracking + reporting are accurate.

A) Intake (Day 0)
Collect from client:
1) Business name + hours
2) Service categories + service area zip codes
3) Primary goal: book calls vs book appointments vs quote capture
4) Booking link (Calendly or similar) OR dispatcher number
5) Lead sources: web form URL, FB lead form, CRM
6) Compliance notes (opt-in language, disclaimers)

B) Script Setup (Day 0–1)
Create a short qualification path:
• Q1 urgency
• Q2 service needed
• Q3 location
• Q4 best time / booking prompt
Rules:
• Keep to 3–5 questions max
• Offer multiple-choice whenever possible
• Always include an “agent handoff” fallback

C) Integration (Day 1)
Choose integration method:
• Webhook/Zapier from form/CRM
• FB lead ads integration
Confirm required fields:
• first name
• phone
• service type (if available)
• source

D) Routing + Handoff (Day 1–2)
Define outcomes:
• Qualified → booking link OR notify team
• Not qualified → polite close + optional alternative offer
• No response → follow-up sequence (e.g., 2 nudges over 24 hours)

E) QA Checklist (Before Launch)
• Test lead comes through within 60 seconds
• SMS content correct and branded
• Booking/handoff works end-to-end
• Lead status updates correctly (New → Contacted → Qualified → Booked)
• Opt-out language present (e.g., “Reply STOP to opt out”)

F) Launch (Day 2–3)
• Turn on live routing
• Monitor first 10 leads
• Adjust questions if drop-off is high

G) Ongoing Optimization (Monthly)
• Review KPI report
• Identify bottlenecks: contact rate, qualified rate, booked rate
• A/B test first message + first question
• Tune business-hour logic and after-hours handling
