STOP/HELP/START IMPLEMENTATION — VERIFICATION EVIDENCE PACK (AGENCY CHECKLIST)

Goal: produce auditable proof that opt-outs are honored and that HELP provides compliant info.

A) Pre-test setup
1. Ensure the outbound sender (Messaging Service/number) is active.
2. Ensure suppression list exists (global per customer at minimum).
3. Ensure inbound webhook is connected to message processor.

B) Test matrix (perform from a real mobile device)
1) STOP keyword handling
- Send: “STOP”
- Expected: immediate confirmation message (one-time) such as:
  “You’re opted out and will no longer receive texts. Reply START to resubscribe. Help: agent_bob_replit+lead-copilot@agentmail.to”
- Expected system state: recipient set to OPTED_OUT=true
- Expected enforcement: any future outbound attempt is blocked (no send) until START

2) Variant opt-outs
- Send: “Stop”, “STOP ” (with trailing space), “UNSUBSCRIBE”, “CANCEL”, “END”, “QUIT”
- Expected: same as above, with normalization applied

3) HELP keyword handling
- Send: “HELP”
- Expected response:
  “Local Lead Response Copilot support: agent_bob_replit+lead-copilot@agentmail.to. Reply STOP to opt out. Msg&data rates may apply.”

4) START resubscribe
- After STOP, send: “START”
- Expected response:
  “You’re re-subscribed. Reply STOP to opt out. Help: agent_bob_replit+lead-copilot@agentmail.to”
- Expected system state: OPTED_OUT=false and consent_status updated

C) Required log evidence (copy/paste format)
For each inbound message above, capture a log entry including:
- timestamp_utc
- customer_id
- from_e164
- to_e164
- inbound_body_raw
- inbound_body_normalized
- keyword_classification (STOP/HELP/START/OTHER)
- action_taken (SET_OPT_OUT / SEND_HELP / SET_OPT_IN)
- suppression_state_before
- suppression_state_after
- outbound_message_id (if any)

D) Screenshot evidence
- Screenshot 1: phone sending STOP
- Screenshot 2: opt-out confirmation received
- Screenshot 3: phone sending HELP
- Screenshot 4: HELP response received
- Screenshot 5: phone sending START
- Screenshot 6: START confirmation received

E) Pass/Fail criteria
FAIL if:
- STOP does not immediately suppress future outbound sends
- HELP response missing support contact or STOP instruction
- Multiple recurring messages are sent after STOP
