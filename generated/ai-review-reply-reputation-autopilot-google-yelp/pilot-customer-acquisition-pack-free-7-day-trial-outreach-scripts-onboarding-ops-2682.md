## Internal Ops Workflow (so we can fulfill pilots immediately)
1) **Ingest**: Review notification arrives in AgentMail inbox.
2) **Triage** (2 minutes): Identify client + location + platform + star rating (if available). If unknown, reply asking which location/platform.
3) **Sentiment + triggers**:
   - Escalate if <3 stars OR negative sentiment OR keywords: “refund”, “lawsuit”, “attorney”, “health department”, “food poisoning”, “injury”, “scam”, “fraud”, “insurance”, “HIPAA”, “unsafe”.
4) **Draft**:
   - Positive: thank + specific detail + invite return.
   - Neutral: thank + acknowledge + invite offline.
   - Negative: apologize (non-admitting), invite offline, no policy violations, no incentives.
5) **Send approval email** with:
   - Proposed reply text
   - Approve / Request edit / Reject
   - Posting link instruction: “Post on Google/Yelp here: {{link}}” (manual link-out)
6) **Log**:
   - Timestamp draft sent, approval status, escalation status.
7) **SLA**:
   - If escalated: email escalation contact immediately.
   - If any unapproved draft > X hours: send reminder.
8) **Weekly KPI** (every Monday): new reviews count, avg rating trend, response time, % responded, negatives, escalations, oldest unresolved.

This package is sufficient to start outreach and onboard pilots today while the MVP UI/automation is finalized.
