# Fulfillment Scope (what we do, what we don’t)
## Review sources covered (week 1)
- **Google Business Profile reviews** (primary)
- **Yelp reviews** (secondary)

## Response & posting method (no API dependency)
Because Google/Yelp APIs and permissions can be limited, we deliver using **human-in-the-loop**:
- Preferred: client grants **GBP Manager** access and **Yelp Business Owner/Manager** access.
- Fallback (lowest friction): **client-post workflow** — we draft responses in a shared doc/dashboard; client posts daily (copy/paste). We still track response time and completion.

## SLA & approval rules
- **4–5★ reviews:** auto-draft + auto-approve unless flagged by banned terms.
- **1–3★ reviews:** always escalated to approval queue; we provide 2–3 response options.
- **Hard flags (always require approval):** medical/legal claims, pricing disputes, threats, discrimination, privacy/PHI, requests for compensation language.

## Escalation rules (negative reviews)
- Create an escalation ticket when: 1–3★ OR contains keywords (refund, lawsuit, scam, rude, waited, injured, fraud, health, BBB).
- Ticket includes: proposed public reply, suggested offline resolution script, owner/staff assignee, due date.
- Track status: New → Awaiting owner input → Posted → Resolved/Unresolved.

## Weekly KPI report (sent every Monday)
- New reviews (Google/Yelp)
- Average rating + change vs last week
- Response rate %
- Median response time
- Negative review count + escalation outcomes
- Top themes (3 bullets)
- Next-week actions (3 bullets)

## Explicit exclusions (protects deliverability + policy)
- No review gating, no paying/incentivizing reviews, no fake reviews.
- No guarantee to remove reviews.
- We can draft dispute text, but client submits disputes from their own accounts.
