# 4) Fulfillment scope (week-1 deliverable, no API dependency)
## Sources covered
- **Google Business Profile reviews** (GBP)
- **Yelp reviews** (Yelp for Business)

## Posting method (deliverable without engineering)
**Default:**
- Client adds us as **Manager** on GBP.
- Yelp: client grants **Business Owner/Manager access** where possible.

**Fallback (always available): “Client-post” workflow**
- We draft responses in a shared doc/portal and send “ready-to-post” links/copy.
- Client posts natively. We track status.

## SLA promises (what we can fulfill immediately)
- New review detected → response drafted within **12h (Pro)** or **24h (Growth/Agency)**.
- Posting occurs via access or client-post same day.

## Escalation rules (brand-safe + risk control)
Auto-approve can be enabled only after onboarding:
- **4–5★ reviews:** can be auto-approved/posted if no flagged keywords.
Require approval + escalation ticket:
- **1–3★ reviews**
- Mentions: refunds, legal threats, discrimination, harassment, HIPAA/medical specifics, pricing disputes, staff names, safety incidents.
Escalation ticket includes: review link, summary, suggested next step, proposed response options (public + optional private).

## Weekly KPI report (sent every Monday)
Metrics:
- New reviews (Google/Yelp)
- Average rating trend (7/30 day)
- Response rate %
- Median response time
- Negative reviews count (1–3★)
- Escalations: opened vs resolved
- Top themes (service, wait time, price, staff, cleanliness)
- “Next week focus” (1–3 bullet actions)
