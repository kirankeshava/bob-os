5) FULFILLMENT SCOPE (WEEK-1 DELIVERABLE) + SLAs
Review sources supported immediately:
- Google Business Profile reviews (via manager access OR client provides screenshots/alerts)
- Yelp reviews (via Yelp Business Owner access OR client-post workflow)

Posting methods (no API dependency required):
- Preferred: client grants GBP Manager access; we draft + post natively.
- Yelp: if access granted, we draft + post; if not, we draft in a shared doc/email and client posts.

Response SLA definition:
- SLA clock starts at the next business hour after review is detected.
- “Responded” means posted to platform OR delivered to client for posting (client-post mode).

Escalation rules (auto-flag to owner contact):
- Any 1–3★ review
- Keywords: refund, lawsuit, attorney, discrimination, injury, HIPAA/PHI, scam, fraud
- Any reviewer claims a specific staff name + misconduct
- Any request for private info (phone, address, medical details)
Escalation deliverable: a draft public reply + a private response suggestion + recommended next action.

Approval rules (human-in-the-loop):
- Auto-approve/post: 4–5★ reviews unless they contain a flagged keyword.
- Require client approval: all 1–3★ reviews and any flagged keyword review.

Weekly KPI report (sent every Monday, covers prior 7 days):
- New reviews count (Google/Yelp)
- Average rating (current + 7-day delta)
- Response rate (% responded)
- Median response time
- Negative reviews (1–3★) count
- Escalations opened vs resolved
- Top 3 themes (service, pricing, scheduling, etc.)
- Action list (3 bullets)

Explicit constraints / compliance boundaries:
- No review gating or incentivizing reviews.
- No impersonation; replies are written as the business voice.
- No deletion guarantees; we do not promise removal of reviews.
- We can suggest dispute language and steps, but client must submit disputes unless explicitly authorized.
