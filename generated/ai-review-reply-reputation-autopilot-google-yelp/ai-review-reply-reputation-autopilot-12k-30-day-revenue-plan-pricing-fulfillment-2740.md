- Set rules: auto-approve criteria and escalation triggers.

2) Daily Review Capture (Mon–Fri)
- Source: GBP/Yelp notifications + manual check once/day (no API required).
- Log new reviews in tracker: date, platform, rating, reviewer name, text, risk flags.

3) Draft Responses
- 4–5★: respond with gratitude + specific reference + invitation to return.
- 1–3★: acknowledge, apologize (if appropriate), invite offline resolution, avoid admissions of liability, avoid incentives.

4) QA + Approval Rules
- Auto-post (or auto-send-to-client) for 4–5★ unless flagged.
- Require client approval for: 1–3★, legal/medical keywords, discrimination claims, refund demands >$X, threats.

5) Posting Method
- Preferred: we post directly (GBP Manager, Yelp access).
- Fallback: send “Ready-to-post” copy with exact steps and deep links.

6) Escalation Handling
- Create escalation ticket, notify contact within SLA.
- Gather facts from client; draft revised response.
- Track status: Open → Awaiting Client → Posted → Resolved.

7) Weekly KPI Report (every Monday)
- One-page summary + metrics + top 3 themes + recommended operational fix.
