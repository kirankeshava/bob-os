FULFILLMENT SCOPE (WEEK-1 DELIVERABLE; NO API REQUIRED)
Review sources:
- Google Business Profile reviews (GBP)
- Yelp reviews (if accessible)

Access + posting method (API-avoidant by design):
- Default for Google: require GBP Manager access (preferred) OR client-post workflow.
- Yelp: preferred = Yelp Business Owner access; fallback = client-post workflow.
Client-post workflow means: we draft replies in a shared doc/email; client posts natively; we track completion.

Daily workflow (operator checklist):
1) Capture new reviews:
- Manual check 1–2x/day OR client turns on review alert emails and forwards to agent_bob_replit+review-bot@agentmail.to.
2) Draft response:
- Use brand voice rules; match platform policy; no incentives; no admissions of liability; no PHI/medical claims.
3) QA + safety scan:
- Check banned phrases, discounts, legal/medical keywords, confidentiality, and tone.
4) Approval rules:
- Auto-approve 4–5★ unless flagged keywords.
- Require client approval for 1–3★, refund threats, legal terms, safety incidents, discrimination, medical, children, or requests for private data.
5) Post:
- Post directly if access granted; otherwise send “Ready-to-post” reply with exact copy + link to review.
6) Escalation ticket:
- Log negative review, suggested offline resolution steps, owner contact, and target response time.
7) Weekly KPI report:
- Send every Monday: last 7 days metrics + action items.

Escalation rules (examples):
- 1★ review => escalate immediately; draft empathetic response + request offline contact; never argue.
- Allegations of harm/safety => escalate; require approval; keep response minimal and defer to offline.
- Refund request => escalate; response acknowledges and moves offline.
- Medical/legal keywords => escalate; avoid claims; refer to client policy.
